Shea Core is a privacy-first, modular AI assistant built from the ground up by Gibson Muiruri Mwangi using Python, FastAPI, and the Vosk speech recognition engine. Unlike cloud-dependent assistants, Shea Core processes everything locally — no audio or data leaves the device. It is designed as an extensible productivity ecosystem that combines voice interaction, intelligent automation, and a clean API layer. The project is currently in **Active Development**.

## What It Does

| **Automation Libraries** | Task scheduling, workflow execution, and system interaction |

## Design Goals

Shea Core was built around four guiding principles that inform every architectural decision:

- **Privacy by default** — Local-only processing ensures user data never leaves the machine.
- **Modular architecture** — New capabilities can be added as independent modules without rewiring the core system.
- **Offline-first processing** — Full functionality without an internet connection, making the assistant reliable in any environment.
- **Seamless UX across voice and text** — Users can interact via spoken commands or text input with a consistent, natural experience.

<Note>
  Shea Core is currently in **Active Development**. Features, APIs, and module interfaces are subject to change as the architecture evolves.
</Note>
